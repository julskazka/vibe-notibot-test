# Скилл: Деплой на Netlify (HTML Kit)

Ты — помощник по публикации статических сайтов. Для HTML Kit деплой особенно простой — нет сборки, нет npm.

---

## Шаг 1. Проверка готовности проекта

Перед деплоем убедись:

1. Файл `index.html` существует в корне проекта
2. Проект залит на GitHub (проверь `git remote -v`)

Если `index.html` нет → скажи пользователю: *"Сначала создайте лендинг — напишите 'создай лендинг'"*
Если GitHub не подключён → запусти skill `github` для пуша

---

## Шаг 2. Инструкция по настройке Netlify

Сообщи пользователю:

> *"Отлично! Проект готов к публикации. Выполните следующие шаги:*
>
> 1. Откройте [app.netlify.com](https://app.netlify.com) и войдите через **GitHub**
> 2. Нажмите **Add new site → Import an existing project**
> 3. Выберите **GitHub** и авторизуйте Netlify
> 4. Найдите и выберите ваш репозиторий с проектом
> 5. В настройках сборки **оставьте все поля пустыми** (для HTML-сайта сборка не нужна):
>    - Build command: *оставить пустым*
>    - Publish directory: `.` (точка — корень папки)
> 6. Нажмите **Deploy**
> 7. Через 30 секунд сайт будет опубликован — пришлите мне ссылку (`.netlify.app`)"*

---

## Шаг 3. Валидация после деплоя

Получив ссылку от пользователя, проверь и сообщи:

✅ Что проверить вручную:
- Открывается ли сайт без белого экрана
- Работает ли Notibot Bridge (кнопки должны реагировать внутри Notibot-приложения)
- Нет ли ошибок в консоли браузера

💡 Совет по домену:
> *"Вы можете изменить случайное имя сайта на красивое: в Netlify перейдите в **Site configuration → Site details → Change site name** и задайте имя вида `my-vibe-app.netlify.app`"*

---

## Шаг 4. Автоматические обновления

Объясни пользователю важное преимущество:

> *"Теперь каждый раз когда вы делаете `git push`, сайт автоматически обновляется на Netlify через 1-2 минуты. Это называется Continuous Deployment."*
