# 📐 Правила проекта: Vibe HTML Kit

---

## 🚀 Протокол 1: Инициализация (при первом открытии проекта)

При каждом старте работы агент **сначала проверяет состояние проекта** и при необходимости настраивает его автоматически. Пользователю ничего делать не нужно.

### Шаг A. Проверить Git

```bash
git status
```

**Если Git не инициализирован** (ошибка "not a git repository"):

```bash
git init
git add .
git commit -m "Initial commit: Vibe HTML Kit"
```

Сообщи пользователю: *"✅ Git инициализирован. Локальная история изменений включена."*

**Если Git уже есть** — пропустить этот шаг.

### Шаг B. Установить pre-commit hook (аудит перед коммитами)

Проверить существует ли `.git/hooks/pre-commit`:

```bash
ls .git/hooks/pre-commit
```

**Если файл не существует** — установить автоматически:

```bash
sh scripts/setup-hooks.sh
```

Сообщи пользователю: *"✅ Автопроверка кода подключена. Теперь агент будет проверять правила перед каждым сохранением в Git."*

**Если файл уже существует** — пропустить этот шаг.

### Шаг C. Только после выполнения A и B — приступить к задаче пользователя.

## ✅ Протокол 2: Проверка после каждой задачи

**После выполнения ЛЮБОЙ задачи** (создал компонент, изменил файл, добавил функцию) — агент **автоматически** запускает аудит:

```bash
node scripts/audit.js
```

### Если аудит выдал ❌ критичные нарушения:
1. **Не говори пользователю "готово"** — сначала исправь нарушения
2. Исправь каждое нарушение согласно правилам (секция ниже)
3. Запусти аудит снова
4. Только после ✅ — сообщи пользователю что задача выполнена

### Если аудит выдал ⚠️ предупреждения:
- Можно сообщить пользователю о выполнении задачи
- Дополнительно сообщи: *"⚠️ Есть небольшие замечания: [перечисли]. Могу исправить сейчас — хочешь?"*

### Если аудит выдал ✅:
- Сообщи пользователю что задача выполнена и код проверен

## 🚫 Правило 1: Лимит размера файла — 150 строк

**Ни один файл не должен превышать 150 строк.**

- Приближается к 150 строкам → извлеки часть логики в новый файл
- Не спрашивай пользователя — просто создай файл и сообщи об этом

```
❌  index.html на 400 строк
✅  index.html < 40 строк + компоненты в js/components/
```

**Как исправить FILE_SIZE:**
1. Найди логически отделяемые функции в нарушающем файле
2. Создай новый файл с правильным именем в нужной папке
3. Перенеси код, обнови импорты

---

## 🚫 Правило 2: Никаких секретов в коде

Запрещено записывать в `.js` или `.html`:
- API ключи и токены
- Пароли
- Приватные URL с встроенными секретами

**Как исправить SECRET_IN_CODE:**
1. Перенеси значение в `.env`
2. Добавь заглушку в `.env.example`
3. Замени в коде на описание откуда брать значение (комментарий)
4. Убедись что `.env` есть в `.gitignore`

---

## 🚫 Правило 3: Точечные правки — не перезаписывать файлы целиком

Перед изменением файла:
1. Прочитай файл полностью
2. Найди конкретный участок
3. Измени **только его**

```
❌  Заменить весь quiz.js из-за правки одной кнопки
✅  Найти функцию handleAnswer() и изменить только её
```

---

## 🚫 Правило 4: Сначала проверь utils.js

Перед созданием новой функции:
1. Открой `js/utils.js`
2. Нет похожей → добавь туда
3. Есть похожая → расширь её

---

## 🚫 Правило 5: Компоненты — изолированы и автономны

```javascript
// ✅ Данные через параметры
export function renderQuizCard({ question, options, onAnswer }) { ... }

// ❌ Данные из глобала
export function renderQuizCard() {
  const q = window.quizData.current; // запрещено
}
```

---

## 🚫 Правило 6: index.html — только разметка

В `index.html` запрещено писать JavaScript (кроме `<script type="module" src="...">`).

## 🛠️ Технический стек

| Технология | Подключение |
|---|---|
| HTML5 | нативно |
| Tailwind CSS | CDN `<script>` в `<head>` |
| Lucide Icons | CDN `<script>` в `<head>` |
| Google Fonts | `<link>` в `<head>` |
| ES Modules | `<script type="module">` |

> Для запуска ES Modules локально нужен dev-сервер.
> Запускай: `npm start` (использует `npx serve .`)
