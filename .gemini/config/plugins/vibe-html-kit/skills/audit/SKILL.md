# Скилл: Аудитор проекта

> [!IMPORTANT]
> Запускай аудит **перед каждым коммитом** и **после добавления новых файлов**.
> Коммит без пройденного аудита — нарушение правил проекта.

---

## Когда запускать аудит

| Ситуация | Обязательно? |
|---|---|
| Перед `git commit` | ✅ Всегда |
| После создания нового компонента | ✅ Да |
| После крупного рефакторинга | ✅ Да |
| После выполнения любой задачи пользователя | ✅ Да |
| По явному запросу пользователя | ✅ Да |

---

## Как запустить

```bash
node scripts/audit.js
```

Или через npm:
```bash
npm run audit
```

---

## Что проверяет аудит

| Проверка | Что считается нарушением |
|---|---|
| **FILE_SIZE** | Файл больше 150 строк |
| **SECRET_IN_CODE** | API ключ / токен в `.js` или `.html` файле |
| **NOTIBOT_OUTSIDE_BRIDGE** | `window.notibot` вызван не из `bridge.js` |
| **LOGIC_IN_HTML** | Inline `<script>` с логикой в `index.html` |
| **ORPHAN_COMPONENT** | Компонент в `components/` не подключён в `app.js` |
| **ENV_NOT_GITIGNORED** | `.env` файл не добавлен в `.gitignore` |

## Как читать результаты

### ✅ Всё чисто — коммитить можно
```
✅ Всё отлично! Нарушений не найдено.
```

### ⚠️ Предупреждения — коммитить можно, но лучше исправить
```
⚠️ Предупреждения (1):
  • [ORPHAN_COMPONENT] js/components/old-widget.js
    Компонент не импортирован в js/app.js.
```

### ❌ Критичные нарушения — коммит заблокирован
```
❌ Критичные нарушения (2) — коммит заблокирован:
  • [FILE_SIZE] js/components/quiz.js
    287 строк (лимит: 150). Извлеки логику в отдельный файл.
  • [SECRET_IN_CODE] js/config.js
    Обнаружен возможный секрет (API ключ / токен).
```

## Алгоритм агента при нарушениях

### При FILE_SIZE:
1. Открой нарушающий файл
2. Найди логически отделяемые части (функции, компоненты)
3. Создай новый файл с правильным именем
4. Перенеси код
5. Обнови импорты
6. Запусти аудит снова

### При SECRET_IN_CODE:
1. Найди строку с секретом
2. Перенеси значение в `.env` (создай файл если нет)
3. Добавь переменную-заглушку в `.env.example`
4. Замени в коде на `process.env.ИМЯ_ПЕРЕМЕННОЙ` или объясни пользователю
5. Проверь `.gitignore` — `.env` должен быть там

### При NOTIBOT_OUTSIDE_BRIDGE:
1. Найди вызов `window.notibot.*` в нарушающем файле
2. Перейди в `js/bridge.js` и создай там функцию-обёртку
3. Замени прямой вызов на импорт из bridge.js

### При ORPHAN_COMPONENT:
1. Открой `js/app.js`
2. Добавь `import` нового компонента
3. Вызови функцию рендера в нужном месте

## Установка автоматического pre-commit hook

Чтобы аудит запускался **автоматически при каждом `git commit`**:

```bash
npm run hooks
```

После этого git будет блокировать коммит если аудит не пройден.

> [!TIP]
> Рекомендуй пользователю запустить `npm run hooks` сразу после `git init`.
> Объясни: это защита проекта от случайного коммита плохого кода.
