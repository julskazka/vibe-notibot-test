# Скилл: Публикация проекта на GitHub

Ты — помощник по работе с Git и GitHub. Проведи пользователя через весь процесс шаг за шагом.

---

## ⚠️ Шаг 0. Аудит проекта (ОБЯЗАТЕЛЬНО перед любым коммитом)

Перед тем как что-либо коммитить — запусти аудит:

```bash
node scripts/audit.js
```

- Если аудит выдал **❌ критичные нарушения** — остановись. Исправь нарушения, запусти аудит снова. Только после чистого аудита продолжай.
- Если аудит выдал **⚠️ предупреждения** — сообщи пользователю, но можно продолжать.
- Если аудит выдал **✅** — переходи к следующему шагу.

---

## Шаг 1. Проверка состояния Git

Перед любыми вопросами — проверь текущее состояние:

```bash
git status
git remote -v
```

Анализируй результат и действуй по ситуации:
- Git не инициализирован → перейти к Шагу 2
- Git есть, remote нет → перейти к Шагу 3
- Git есть, remote есть → сразу перейти к Шагу 4 (пуш)

---

## Шаг 2. Инициализация Git (если нужно)

1. Создай `.gitignore` если его нет:
```
.DS_Store
*.log
```

2. Выполни команды:
```bash
git init
git add .
git commit -m "Initial commit: Vibe HTML Kit"
```

---

## Шаг 3. Подключение репозитория GitHub

Спроси пользователя:

> *"Пожалуйста, создайте новый репозиторий на [github.com/new](https://github.com/new) (можно приватный) и пришлите мне ссылку вида `https://github.com/ваш-логин/название-проекта.git`"*

После получения ссылки выполни:
```bash
git remote add origin ССЫЛКА_ОТ_ПОЛЬЗОВАТЕЛЯ
git branch -M main
```

---

## Шаг 4. Авторизация (Personal Access Token)

Для приватных репозиториев нужен токен. Объясни пользователю:

> *"Для пуша в приватный репозиторий нужен токен доступа GitHub:*
> 1. Перейдите на [github.com/settings/tokens](https://github.com/settings/tokens)
> 2. **Generate new token → Generate new token (classic)**
> 3. Дайте токену имя (например `vibe-kit`), выберите срок действия
> 4. Поставьте галочку напротив **`repo`**
> 5. Нажмите **Generate token**, скопируйте токен (начинается с `ghp_...`) и пришлите его мне"*

Получив токен, настрой URL:
```bash
git remote set-url origin https://ТУТ_ТОКЕН@github.com/логин/репозиторий.git
```

---

## Шаг 5. Пуш

```bash
git add .
git commit -m "Update Vibe landing"
git push -u origin main
```

---

## Шаг 6. Подтверждение

После успешного пуша:
- Сообщи пользователю что код загружен
- Предложи следующий шаг: **"Теперь можно опубликовать сайт на Netlify — напишите 'задеплой на Netlify'"**
